# 维护说明

这份文档的目标不是讲历史，而是明确这个仓库现在到底怎么维护、优先维护什么、哪些事情会继续做。

## 这个分支的定位

`LizzieYzy Next-FoxUID` 不是一次性修补版，而是一个面向继续维护的分支。

当前维护重点很明确：

- 修复原版已经失效的野狐棋谱同步
- 把输入方式统一成普通用户更容易理解的 **野狐ID / Fox ID**
- 继续提供更好安装的多平台发布包
- 尽量降低新用户第一次使用的门槛
- 在保留 LizzieYzy 原有能力的前提下，优先保证“能装、能开、能抓谱、能分析”

## 当前优先维护的范围

### 1. 野狐棋谱获取

优先保证下面这条链路稳定：

1. 用户打开程序
2. 进入“野狐棋谱（输入野狐ID获取）”
3. 输入纯数字野狐ID
4. 成功获取最新公开棋谱

和这条主链路冲突的旧逻辑，优先服从当前维护策略。

### 2. 多平台发布包

当前发布包策略以“普通用户能快速选对并直接用”为核心。

优先维护这些包：

- Windows 64 位：`with-katago`、`without.engine`
- macOS Apple Silicon：`.dmg`
- macOS Intel：`.dmg`
- Linux 64 位：`with-katago`
- 进阶用户包：`Macosx.amd64.Linux.amd64.without.engine.zip`

### 3. 内置 KataGo 整合体验

整合包的目标是：

- 带上合适的引擎与权重
- 第一次启动尽量自动识别内置 KataGo
- 少让用户手动填引擎路径

如果某个改动会影响这个目标，需要优先评估用户体验再合并。

## 当前统一术语

为了避免 README、发布页、界面、issue 模板各说各话，现在统一使用以下口径：

- 中文：`野狐ID`
- 英文：`Fox ID`
- 包类型：`with-katago`、`without.engine`

不再把用户入口写成“用户名搜索”或只写“UID”。

## 发布维护原则

### 1. 用户先看懂，再下载

发布页要做到用户一眼能理解：

- 哪个包适合自己系统
- 哪个包是开箱即用
- 哪个包需要自己配引擎

### 2. 少而清楚，不要包型泛滥

目前已经明确收缩的方向：

- 不再保留多余的 macOS `.app.zip`
- 不再保留“other systems”这种用途不清晰的旧包
- Windows 64 位必须同时兼顾开箱即用用户和自定义引擎用户

### 3. Release Notes 的第一句要讲清价值

每次发版，最前面都应该先写最核心的用户价值，例如：

- 原版野狐棋谱同步已失效，这个维护版已修复
- 输入野狐ID即可获取最新公开棋谱
- 新增或恢复了哪些可直接使用的发布包

## 维护者建议遵守的检查项

每次改动下面这些内容时，请至少补一轮自查：

- 野狐抓谱逻辑
- 包内引擎路径
- 权重文件位置
- 发布包文件名
- README 下载建议
- 界面和文档中的 Fox ID / 野狐ID 术语

## 什么问题最值得优先修

如果维护时间有限，优先级建议如下：

1. 无法启动
2. 无法抓取野狐公开棋谱
3. 发布包内容和 README 描述不一致
4. 整合包内置引擎失效
5. 明显误导用户的界面或文案
6. 其他体验优化

## 现在仍然欢迎的贡献方向

- 实机安装验证
- 发布包验收记录
- 多语言文档完善
- 野狐接口兼容性反馈
- 打包流程稳定性修复
- 首页视觉和演示素材优化
